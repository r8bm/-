# Delivery Tracking System — Driver APK + Restaurant Control APK

هذا المشروع هو حزمة تطبيقات دلفري حسب المواصفات المطلوبة:

- تطبيق مندوب Android APK.
- تطبيق مطعم/إدارة Android APK.
- Backend API للحسابات والتتبع والعقوبات والمحاسبة.
- حساب وقت الرجوع حسب المسافة + وقت السماح.
- حساب سعر الطلب = سعر الطلب + أجرة التوصيل.
- تحديد عمولة المندوب من إعدادات المطعم.
- عقوبة خصم نسبة من العمولة لمدة يحددها المطعم، والنسبة المخصومة تذهب للمطعم.
- المحاسبة: المندوب يسلم كامل مبلغ الزبون للمطعم، ثم المطعم يدفع العمولة النهائية بعد العقوبات.
- كشف حساب يومي يشمل عدد الطلبات، الأسعار، التوصيل، العمولة، عدد العقوبات، وقت العقوبات والخصومات.
- موافقة قانونية داخل تطبيق المندوب على تتبع الموقع والكاميرا والتتبع بالخلفية.
- صلاحيات Android المطلوبة فقط: إنترنت، موقع، موقع بالخلفية، Foreground Service، كاميرا، إشعارات.

> ملاحظة مهمة: بيئة ChatGPT الحالية لا تحتوي Flutter أو Android SDK، لذلك لا يمكن بناء APK فعلي داخلها. الكود مرفق كاملًا مع سكربتات بناء APK تعمل على أي جهاز فيه Flutter أو عبر GitHub Actions.

## بنية المشروع

```text
backend/
  Node.js Express API + JSON database بدون Native dependencies

apps/driver_app/
  كود تطبيق المندوب Flutter

apps/restaurant_app/
  كود تطبيق المطعم Flutter

scripts/
  سكربتات تجهيز Flutter وبناء APK

.github/workflows/build-apks.yml
  Workflow لبناء APKs تلقائيًا من GitHub Actions

docs/
  مستندات الخصوصية والنشر والمتطلبات
```

## تشغيل السيرفر Backend

```bash
cd backend
cp .env.example .env
npm install
npm run reset-db
npm start
```

إذا كان المنفذ 8080 مستخدمًا:

```bash
PORT=8099 npm start
```

اختبار الصحة:

```bash
curl http://localhost:8080/health
```

بيانات الدخول التجريبية:

```text
Admin:  07711111111 / 123456
Staff:  07733333333 / 123456
Driver: 07722222222 / 123456
```

## بناء APK على جهازك

المتطلبات:

- Flutter stable مثبت.
- Android Studio أو Android SDK.
- Java JDK.

من جذر المشروع:

```bash
./scripts/build_all_apks.sh
```

الناتج سيكون هنا:

```text
dist/delivery-driver.apk
dist/delivery-restaurant.apk
```

لبناء تطبيق واحد فقط:

```bash
./scripts/build_driver_apk.sh
./scripts/build_restaurant_apk.sh
```

---

## بناء APK عبر GitHub Actions

1. ارفع المشروع إلى GitHub.
2. افتح تبويب Actions.
3. شغل Workflow باسم: `Build Delivery APKs`.
4. بعد انتهاء البناء، حمل Artifact باسم `delivery-apks`.

في شاشة الدخول يوجد حقل `رابط السيرفر`.

على محاكي Android استخدم:

```text
http://10.0.2.2:8080
```

## منطق الوقت والعقوبات

عند ضغط المندوب `تم التسليم`، يحسب النظام:

```text
وقت الرجوع المسموح = وقت الطريق المتوقع حسب المسافة والسرعة + وقت السماح من إعدادات المطعم
```

ثم عند ضغط `رجعت للمطعم`:

- يتحقق من أنه داخل نطاق المطعم Geofence.
- يحسب دقائق التأخير.
- إذا وجد تأخير، ينشئ عقوبة `pending` أو `active` حسب إعدادات المطعم.
- العقوبة تخصم نسبة من عمولات الطلبات التي تقع داخل وقت العقوبة.

## المحاسبة

النظام يعتمد الطريقة التي طلبتها:

```text
المندوب يسلم كامل مبلغ الطلب للمطعم.
ثم المطعم يعطيه العمولة النهائية بعد احتساب العقوبات.
```

كشف الحساب اليومي يحسب:

- عدد الطلبات.
- إجمالي سعر الطلبات.
- إجمالي أجور التوصيل.
- إجمالي المبالغ الكاش المستلمة.
- المبلغ الذي يجب أن يسلمه المندوب للمطعم.
- العمولة قبل العقوبات.
- عدد العقوبات.
- وقت العقوبات.
- إجمالي الخصم.
- العمولة النهائية.
- المبلغ المتبقي على المندوب إن لم يسلم كامل الكاش.
